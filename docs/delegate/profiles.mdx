# Profiles & Configuration

The delegate's behavior is controlled by a **profile**. Profiles define how much the delegate decides, how often it interrupts, how strict it is, and when it escalates to you.

---

## Profile Comparison

| Profile | Autonomy | Critique | Interruptions | Best for |
|---------|----------|----------|---------------|----------|
| `internal_full` | high | strict | minimal | testing Fernsehturm, long autonomous runs |
| `assisted_build` | moderate | moderate | occasional | building features from high-level ideas |
| `review_heavy` | none | strong | frequent | controlled development, human-in-the-loop |
| `minimal` | off | light | rare | full manual control |

---

## Configuration

```bash
fst delegate set-profile assisted_build   # switch profile
fst delegate enable                        # turn on
fst delegate disable                       # turn off
fst delegate status --json                 # inspect current state
```

---

## Choosing a Profile

**Start with `internal_full`** when you want to test Fernsehturm or run a large autonomous task. Let it find the gaps in your setup.

**Use `assisted_build`** for day-to-day feature work. The delegate fills most gaps and suggests decisions without blocking you constantly.

**Switch to `review_heavy`** when you want to stay in control of every major decision. The delegate asks questions rather than answering them.

**Drop to `minimal`** when you know exactly what you want and the delegate would just be noise.

---

## Controlling Noise

If the delegate feels too active:

* switch to a weaker profile
* reduce question frequency in profile settings
* filter low-severity outputs with `fst feedback list --min-severity high`

A useful pattern: start a new feature with `assisted_build`, then drop to `minimal` once the shape is clear.
