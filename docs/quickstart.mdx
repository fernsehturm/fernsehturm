# Quickstart

From zero to a running system. Choose the mode that fits how you want to work.

---

## Two modes

| | Manual | Agentic |
|---|---|---|
| **Who drives the loop** | you, via CLI | the agent, via `/fst` |
| **Delegate role** | reviews your work, flags gaps | guides the agent between steps |
| **Control** | full | supervised automation |
| **Best for** | learning the system, fine-grained work | feature development at scale |

Both modes use the same underlying system. The difference is who calls the commands.

---

## 1. Initialize

Both modes start here:

```bash
fst self init
```

---

## Manual mode

You drive each step. The delegate reviews your work and flags problems—you decide what to do with its feedback.

### 2. Enable the delegate

```bash
fst delegate set-profile review_heavy
fst delegate enable
```

`review_heavy` asks questions rather than making decisions. It's the right profile when you want to stay in control.

### 3. Create a request

```bash
fst request create "Build a hello world HTTP server"
```

### 4. Derive requirements

```bash
fst semantic run derive-requirements --goal <goal-id>
```

Review the proposed requirements alongside delegate feedback:

```bash
fst artifact list --type requirement --json
fst feedback list --type delegate_critique --json
```

Accept what you're satisfied with:

```bash
fst artifact accept --id <req-id>
```

### 5. Create a decision

```bash
fst artifact put \
  --type decision \
  --body "Use Node.js with the built-in http module" \
  --status accepted
```

### 6. Generate code

```bash
fst semantic run generate-code --requirement <req-id>
```

Check delegate feedback before accepting:

```bash
fst feedback list --type delegate_critique --json
fst artifact accept --id <code-id>
```

### 7. Materialize, run, verify

```bash
fst materialize
fst run
fst verify run
```

### 8. Refine if needed

```bash
fst feedback list --unresolved --json
# address issues, then loop from step 6
```

Recurring feedback usually means a missing requirement—fix it upstream, not in the code.

---

## Agentic mode

The agent drives the loop. You describe the goal; the system supervises every step; the delegate guides the agent between them. You only intervene when the system asks.

### 2. Enable the delegate

```bash
fst delegate set-profile assisted_build
fst delegate enable
```

`assisted_build` fills gaps and suggests decisions autonomously. The agent doesn't need to ask you about routine details.

### 3. Install the `/fst` skill

```bash
fst skill add
```

This registers the `/fst` command in the agent and binds it to the supervisor loop.

### 4. Start the task

There are three ways to start. For a first run, use the default:

```text
/fst Build a hello world HTTP server
```

The system explores first, then moves to implementation once it has enough context.

If the goal is already clear, skip exploration:

```text
/fst apply Build a hello world HTTP server
```

To understand the system without making any changes:

```text
/fst explore Build a hello world HTTP server
```

The agent runs the full loop without further input:

```text
→ fst next → derive requirements  → fst step submit
→ fst next → run consistency      → fst step submit
→ fst next → generate code        → fst step submit
→ fst next → materialize          → fst step submit
→ fst next → verify               → fst step submit
→ fst next → finish
```

The delegate critiques each step and feeds results back into the loop. The agent does not decide what comes next—the system does.

### 5. Intervene when asked

If the system hits a decision it cannot resolve, it stops and asks you:

```text
[ask_user] Use JWT or sessions?
```

Answer, and the loop resumes.

### 6. Inspect the result

```bash
fst summary
fst feedback list --unresolved --json
```

If something needs refinement, describe the change and run `/fst` again.
