# Delegate Setup

The delegate is optional but recommended. It acts as a synthetic product partner—critiquing outputs, flagging gaps, and asking clarifying questions between steps. See [What the Delegate Is](../delegate/overview) for a full explanation.

---

## Enable the delegate

```bash
fst delegate enable
```

## Choose a profile

Profiles control how much the delegate decides, how often it interrupts, and how strict it is.

```bash
fst delegate set-profile assisted_build
```

| Profile | Autonomy | Best for |
|---------|----------|----------|
| `internal_full` | high | testing Fernsehturm, long autonomous runs |
| `assisted_build` | moderate | day-to-day feature work |
| `review_heavy` | none | controlled development, manual review |
| `minimal` | off | full manual control |

**Start with `assisted_build`** if you are unsure. You can change the profile at any time.

## Check current state

```bash
fst delegate status --json
```

## Disable

```bash
fst delegate disable
```

---

## Recommended starting configuration

For most projects, this is a good starting point:

```bash
fst delegate set-profile assisted_build
fst delegate enable
```

Drop to `review_heavy` if you want more control. Escalate to `internal_full` when running large autonomous tasks or testing the system.
