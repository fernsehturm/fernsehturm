# FAQ

## Does FST write the code?

No. The agent writes, tests, revises, and proposes work. FST controls whether work is allowed to move forward.

## What is the Accepted Work Context?

It is the recorded starting universe for Exploration.

It answers:

```text
Where may Exploration look?
```

It does not answer:

```text
What may Build change?
```

Build permission comes later from retained scope in the ExplorationNote.

## Why does FST ask me questions?

FST asks when your answer changes scope, behavior, policy, risk, context choice, or materialization.

It should not ask you to micromanage routine implementation details.

## What does "coherent Composition" mean?

It means one exact Composition revision passed the required checks for its pinned members.

It does not mean globally accepted, canonical, best, permanent, or already materialized.

## Why no global accepted state?

Because global acceptance makes parallel work and alternatives fight over one mutable truth.

FST lets several possible worlds exist at once. Each Composition can be checked independently. A change is effective only inside a coherent Composition that includes it.

## Can multiple agents work at the same time?

Yes. Parallel agents can build separate Candidates. Compose checks whether the selected Candidates and direct members hold together as a possible world.

Conflicts are represented as Composition blockers instead of being hidden in file merges.

## What if FST blocks the work?

A blocker is useful output. It tells you what is missing before the work can safely move forward.

Common blockers include missing user evidence, stale Exploration, scope expansion required, missing verification, policy conflict, decision conflict, and implementation conflict.

## What is materialization?

Materialization projects a pinned coherent Composition into a concrete sink: a workspace, patch, generated directory, migration folder, or deployment target.

It is not global acceptance and it is not proof of correctness by itself.

## How do I know the result is trustworthy?

Ask for the final trace:

```text
Accepted Work Context
ExplorationNote and retained scope
Candidate
Composition
gate checks
user evidence
Observations
materialization result
remaining blockers
```

If the result cannot be traced, FST has not delivered the confidence it is meant to provide.
