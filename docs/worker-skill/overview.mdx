# Worker & `/fst` Skill

> **The `/fst` skill turns any AI agent into a Fernsehturm worker that executes only supervised, state-driven actions until completion.**

---

## What Changes

A normal agent plans, explores, and edits code directly. A Fernsehturm worker does none of that.

| Normal agent | Fernsehturm worker |
|---|---|
| decides what to do | asks the system what to do |
| explores freely | executes only approved steps |
| edits code directly | writes everything through the system |
| flexible, open-ended | deterministic, supervised |

---

## Mental Model

```text
Agent (worker) → asks → fst (supervisor) → decides → worker executes → repeat
```

The worker is not autonomous. It is a controlled executor. The system drives everything.

---

## Getting Started

Install the skill:

```bash
fst skill add
```

This is the only required setup. It installs the worker behavior, registers `/fst`, binds the agent to the `fst` CLI, and activates supervised execution.

Start a task:

```text
/fst Add a login endpoint
```

This creates a request, activates the supervisor loop, and puts the agent under control. The worker does not proceed until the system tells it what to do.

---

## What This Enables

Using `/fst`:

* removes agent drift
* enforces consistent workflows
* ensures all changes are traceable
* scales reliably across features
* enables automation without unpredictable behavior
