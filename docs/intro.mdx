# Introduction

Fernsehturm (FST) is a control system for AI-assisted software development.

You keep the product direction. Your agent does the nitty-gritty work. FST keeps the work bounded, traceable, and checked before the agent moves forward.

The headline promise is simple:

> Build Software with AI without Losing Control.

FST delivers that promise by making every meaningful change pass through a controlled path:

```text
Accepted Work Context -> Exploration -> Build -> Compose
```

The agent is free to inspect, experiment, draft, run tests, and revise inside a stage. FST controls the transition between stages.

## What You Do

Most of the time, you only do five things:

1. Say what you want.
2. Confirm the Accepted Work Context when there is more than one sensible starting point.
3. Answer decisions the system cannot safely infer.
4. Approve risky scope changes, policy exceptions, or materialization choices.
5. Review the final result.

You do not manually maintain a requirements database. You do not guide every implementation step. You do not review every line of generated code just to keep the agent from drifting.

## What FST Does For You

FST keeps the agent from turning a loose prompt into uncontrolled changes.

It makes sure the agent starts from the right part of your system, stays inside the agreed scope, stops for decisions that need your judgment, and leaves behind a trace of what was changed and why.

The practical benefit is that you can ask direct questions:

- What did the agent change?
- Why was it allowed to change that?
- Which decisions did I make?
- What was checked?
- What is still unresolved?

FST's job is to make those answers concrete instead of depending on the agent's memory.

## Why This Is Different

Ordinary AI coding merges by files. FST composes by meaning.

That matters because large changes often fail outside the edited file. A session timeout change may touch product behavior, security policy, existing tests, user-visible wording, and old architectural decisions. FST forces those connections into the record before the change is treated as coherent.

The user experience becomes very focused:

```text
You give direction.
FST asks only when the answer changes the safe path.
The agent works inside the recorded path.
```
