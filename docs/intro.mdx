# Introduction

Fernsehturm (FST) is a local control layer for agent work.

Agents can inspect, draft, run tools, and propose effects. FST decides what is
allowed next by evaluating a process profile, recorded artifacts, approvals,
and the current run state.

```text
agent intent -> installed process -> route -> evidence -> controlled effect
```

FST is local-first:

```text
one fst Linux binary
local_file or sqlite store
versioned workspace config
installed process registry
generated fst-* process commands
hook-backed process logic
scenario runner
replayable evidence
local MCP controller
```

## What FST Controls

FST answers these questions for an agent:

- Is this action part of the selected installed process?
- Which gate applies before the action?
- Which artifacts or approvals are missing?
- Should the agent continue, ask, repair, wait, or stop?
- May this effect be mocked, shadowed, or materialized?
- What evidence should be recorded for replay?

The agent stays useful. It does not become the authority.

## What You Do

You install `fst`, run `fst setup`, select the processes you want in the local
workspace, run scenarios, and connect your agent through the local MCP
controller.

Bundled processes include `local_patch_review` for developer patch review and
`purchase_request_preflight` for controlled purchase packets. Each installed
process has its own generated command and its own process contract.
