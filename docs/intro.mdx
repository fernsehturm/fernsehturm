# Introduction

Fernsehturm (FST) helps agents do useful work without letting the agent decide
what is approved, complete, or safe to change.

The short version:

```text
Agents do the work. FST controls what counts.
```

For example, an agent can prepare a purchase request. FST can check that the
request follows the process, records the right evidence, and stops before a real
purchase unless the required approval exists.

## The First Thing To Try

Start with the purchase quickstart. You install FST, start the local agent
interface, ask your agent for a purchase, and watch FST decide whether the
process can continue.

:::note Coming soon
The public installer is not available yet.
:::

```bash
curl -fsSL https://www.fernsehturm.dev/install.sh | bash
export PATH="$HOME/.local/bin:$PATH"

fst setup \
  --workspace "$HOME/fst-workspace" \
  --store sqlite \
  --install-process purchase_request_preflight@0.1.0 \
  --no-password

fst mcp start --workspace "$HOME/fst-workspace"
```

Then start your agent and ask:

```text
Buy me a 4K monitor, around $400, for design work.
Use my purchase process.
```

The first run should show the basic loop:

```text
user request
-> agent prepares the purchase packet
-> FST checks the purchase process
-> FST records the route and evidence
-> agent stops when approval is missing
```

It does not place a real order. It shows how FST keeps an agent inside a
controlled purchase process while the agent still does useful work.

## What FST Adds

Without FST, an agent can say:

```text
I prepared the purchase request and it looks approved.
```

With FST, the process can require:

- a specific process profile
- a controlled run
- typed request details
- required evidence
- an approval path when money or authority is involved
- replay showing why the work was allowed or blocked

The agent stays useful. It does not become the authority.
