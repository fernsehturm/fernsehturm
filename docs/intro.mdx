# Introduction

## What is Fernsehturm?

Fernsehturm is a **database-native software system designed for AI agents**.

Unlike traditional development environments where code lives in files, Fernsehturm stores code, requirements, and decisions as structured artifacts in a SQLite database. Files are generated projections—disposable outputs derived from the database, not the source of truth.

## Why it exists

Software built by AI agents suffers from a fundamental problem: **codebases don't scale in consistency**. As features accumulate, agents lose track of prior decisions, requirements drift from implementations, and the system becomes incoherent.

Fernsehturm addresses this by:

- Giving agents a **structured interface** instead of a text-based file system
- Maintaining **traceability** from goals through requirements to running code
- Enforcing **consistency checks** that catch gaps before they compound
- Providing **structured feedback** that drives iteration rather than patch-work fixes

## Core idea

> Software is a system of **intent, decisions, and artifacts**—not files.

Agents don't need to manage files. They need to express intent, receive critique, and refine proposals until the system is coherent and working.

## The Delegate

On its own, Fernsehturm gives agents structure. The **delegate** gives them guidance.

The delegate is an optional subsystem that acts like a configurable product partner—asking clarifying questions, identifying missing requirements, critiquing implementations, and suggesting improvements. It fills the gap left when there is no human product owner in the loop.

> The delegate is a **configurable, synthetic product partner** that guides the agent—asking questions, pointing out gaps, and critiquing results—without ever becoming the system's authority.

You control how much it does. At full strength it acts like an autonomous product owner. At minimum it stays quiet and only flags critical gaps. In between, it helps you move faster without losing control.

The delegate never enforces rules, makes changes directly, or overrides decisions. It produces structured feedback that the controller interprets—the agent then acts accordingly.
